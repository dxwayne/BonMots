\section*{Support Files for SAS}

This repository is the starting point for feathering the nest with
IRAF command files, Python utilities, and other items to assist with
using IRAF under Anaconda.

The repository should be cloned into
{\color{verbcolor}{\verb#~/iraf#}} directly. If you cloned it
elsewhere into your file system -- you can move it to
{\color{verbcolor}{\verb#~/iraf#}} without upsetting git.

\subsection{home\_iraf}

This subdir ({\color{verbcolor}{\verb#~/iraf/smtsci/home_bin#}})
has:

- {\color{verbcolor}{\verb#.cl#}} scripts that are executed within
iraf by running an instance of {\color{verbcolor}{\verb#cl#}} and
using ``file redirection'' to ``input'' the
{\color{verbcolor}{\verb#~/iraf/smtsci/home_bin/xxx.cl#} script.
- the {\color{verbcolor}{\verb#.dat#}} and {\color{verbcolor}{\verb#.db#}}
files serve as templates to dial IRAF to a particular observatory.

The basic manifest in the package:

is a subset of around 400 parameters available. Here
the the ra/dec and x/y locations of the centers of

real stars from small galaxies and other features

\section{Using ``git''}

The git suite of tools were designed to allow the developers
of the Linux kernal and support routines to collaborate.
It is extremely powerful -- but a proper subset serves
our needs.

I would recommend:

\vspace{-.15cm}
\begin{enumerate}\addtolength{\itemsep}{-0.5\baselineskip}
   \item   take good notes about what you do. It is done infrequently,
and needs special attention to detail.
   \item   Make a directory {\color{verbcolor}{\verb#~/git#}}
   \item   {\color{verbcolor}{\verb#cd#}} there and do all the clones
   \item   use git to create a local repository, say {\color{verbcolor}{\verb#myiraf#}}
   \item   copy the files you want to specialize for your site to {\color{verbcolor}{\verb#myiraf#}}
from {\color{verbcolor}{\verb#~/iraf/smtsci#}} and use git to manage them
\end{enumerate}
